fix: restore CSS sticky stacking cards, remove overflow-x-hidden from body

Root cause: overflow-x-hidden on body in layout.tsx breaks position:sticky
on all descendant elements. The previous JS fixed-positioning workaround
caused reflow oscillation (flashing/glitching near profile section).

Changes:
- Revert cards/index.tsx from JS-controlled fixed positioning back to
  CSS position:sticky -- simpler, no reflow issues
- Remove overflow-x-hidden from body in layout.tsx -- this was the
  single property breaking sticky. overflow-x:clip in globals.css
  already handles horizontal overflow without breaking sticky context
- Remove scroll-snap system from globals.css (was snapping past cards)
- Remove overflow-x-clip from ancestor chain (lazy-section,
  scroll-stack-section, page.tsx main) -- belt-and-suspenders cleanup
- Remove unused stacking-sections.tsx component

Technical note: overflow-x:hidden creates a new scroll container which
prevents sticky from working. overflow-x:clip provides the same visual
clipping without creating a scroll container, preserving sticky behavior.
