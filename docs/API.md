# API

## Public routes

| Route | Purpose |
|---|---|
| `/` | Portfolio homepage with hero, sections, navigation, and contact CTA. |
| `/blog` | Blog index with category filtering. |
| `/blog/[slug]` | Static blog article route generated from MDX content. |
| `/feed.xml` | RSS feed for blog posts. |
| `/sitemap.xml` | Search crawler sitemap generated by `app/sitemap.ts`. |
| `/robots.txt` | Search crawler directives generated by `app/robots.ts`. |

## Navigation contract

The homepage navigation is exported from `components/nav/index.tsx` as `Navigation`. Links are configured in `components/nav/nav-links.ts` and point to homepage section IDs or site routes. The contact link remains a CTA instead of a duplicated mobile list item.

## Environment variables

No runtime secrets are required for local development. Deployment hosts may provide standard Next.js environment variables, but this app does not require committed secret files.

## Package scripts

| Script | Purpose |
|---|---|
| `bun run dev` | Start the Next.js development server. |
| `bun run lint` | Run ESLint across the repository. |
| `bun run test` | Run the Vitest suite. |
| `bun run ci` | Run ESLint, then run the production build. |
| `bun run ship:dev` | Alias for `bun run ci`; use before shipping local development changes. |
| `bun run build` | Run tests through `prebuild`, then build Next.js. |
| `bun run start` | Start the production server after a build. |

## Version source

The root `VERSION` file is the human-readable release source. `package.json` mirrors it for package metadata.
