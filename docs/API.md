# API

## Public routes

| Route | Purpose |
|---|---|
| `/` | Portfolio homepage with hero, sections, navigation, and contact CTA. |
| `/blog` | Blog index with category filtering. |
| `/blog/[slug]` | Static blog article route generated from MDX content. |
| `/tools` | Interactive AI project cost estimator and prompt linter. |
| `/api/prompt-lint` | Rate-limited optional Hugging Face prompt critique endpoint; deterministic local lint runs without secrets. |
| `/feed.xml` | RSS feed for blog posts. |
| `/sitemap.xml` | Search crawler sitemap generated by `app/sitemap.ts`. |
| `/robots.txt` | Search crawler directives generated by `app/robots.ts`. |

## AI tools

The `/tools` page ships two free-first tools: an AI project cost estimator and a system prompt linter. The estimator is deterministic and client-side. The prompt linter always runs deterministic local checks for role, inputs, output format, constraints, success criteria, examples, failure behavior, and tone, then returns a score, recommendations, quick wins, and a rewrite brief. `POST /api/prompt-lint` may add a Hugging Face critique when `HUGGINGFACE_API_TOKEN` is configured on the server. The API is in-memory rate limited to eight requests per minute per forwarded client IP.

## LLM pricing search

The `/llm-prices` search box filters rows by model name and provider slug after trimming the query. Rendered table rows use a composite row key because the upstream pricing feed can repeat model identifiers, and duplicate keys can make React keep stale rows visible after a query narrows the list.

## Navigation contract

The homepage navigation is exported from `components/nav/index.tsx` as `Navigation`. Links are configured in `components/nav/nav-links.ts` and point to homepage section IDs or site routes. The contact link remains a CTA instead of a duplicated mobile list item.

## Environment variables

No runtime secrets are required for local development. Deployment hosts may provide standard Next.js environment variables. `HUGGINGFACE_API_TOKEN` is optional and enables the `/tools` prompt linter AI critique. `HUGGINGFACE_PROMPT_LINTER_MODEL` optionally overrides the default free/serverless Hugging Face model. Do not commit `.env` values.

## Package scripts

| Script | Purpose |
|---|---|
| `bun run dev` | Start the Next.js development server. |
| `bun run lint` | Run ESLint across the repository. |
| `bun run test` | Run the Vitest suite. |
| `bun run ci` | Run ESLint, then run the production build. |
| `bun run ship:dev` | Alias for `bun run ci`; use before shipping local development changes. |
| `bun run build` | Run tests through `prebuild`, then build Next.js. |
| `bun run start` | Start the production server after a build. |
